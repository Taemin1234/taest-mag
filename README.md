# 테이스트 매거진 (Taest Magazine) - 프로젝트 설명서

## 📖 프로젝트 개요

**테이스트 매거진**은 다양한 분야의 에디터가 글을 게시하고 독자가 카테고리별로 탐색할 수 있는 온라인 매거진 플랫폼입니다. "심도보단 감도, 트렌드보단 당신의 결"이라는 컨셉으로 사용자의 취향을 수집하고 공유하는 콘텐츠 기반 웹 서비스를 구현했습니다.

이 프로젝트는 **풀스택 개발**을 통해 백엔드와 프론트엔드를 모두 직접 구현하며, 현대적인 웹 애플리케이션의 구조와 보안, 성능 최적화를 이해하는 것을 목표로 개발되었습니다.

## 🛠 기술 스택 및 사용 목적

### **프론트엔드**

#### **Next.js 15.3.6**
- **사용 목적**: 서버 사이드 렌더링(SSR)과 정적 사이트 생성(SSG)을 통한 SEO 최적화 및 초기 로딩 성능 향상
- **적용 위치**: 
  - 메인 페이지, 게시글 상세 페이지, 에디터 페이지 등 모든 라우팅
  - `Suspense`와 `use()` 훅을 활용한 스트리밍 데이터 페칭으로 사용자 경험 개선
  - 동적 메타데이터 생성으로 소셜 미디어 공유 최적화

#### **React 19.1.0**
- **사용 목적**: 컴포넌트 기반 UI 구축 및 상태 관리
- **적용 위치**: 
  - 재사용 가능한 컴포넌트 구조 (PostList, FeaturePost, EditorInfo 등)
  - 서버 컴포넌트와 클라이언트 컴포넌트 분리로 성능 최적화

#### **TypeScript**
- **사용 목적**: 타입 안정성 확보 및 개발 생산성 향상
- **적용 위치**: 
  - 모든 컴포넌트와 유틸리티 함수에 타입 정의
  - API 응답 타입, 사용자 역할 타입 등 인터페이스 정의

#### **Zustand 5.0.4**
- **사용 목적**: 클라이언트 사이드 전역 상태 관리
- **적용 위치**: 
  - 사용자 역할(role) 상태 관리 (`useRoleStore.ts`)
  - 로그인 상태에 따른 UI 권한 제어

#### **TinyMCE 7.9.1 / @tinymce/tinymce-react 6.2.1**
- **사용 목적**: 리치 텍스트 에디터 구현
- **적용 위치**: 
  - 게시글 작성 및 수정 페이지
  - 이미지 삽입, 텍스트 포맷팅 등 다양한 콘텐츠 편집 기능 제공

#### **DOMPurify 3.2.6 / isomorphic-dompurify 2.26.0**
- **사용 목적**: XSS(Cross-Site Scripting) 공격 방지
- **적용 위치**: 
  - TinyMCE로 작성된 HTML 콘텐츠의 안전한 렌더링
  - 서버 사이드와 클라이언트 사이드 모두에서 HTML 정제

#### **Swiper 11.2.8**
- **사용 목적**: 반응형 이미지 슬라이더 구현
- **적용 위치**: 
  - 메인 페이지의 특별호 게시글 슬라이더
  - 모바일 환경에서의 터치 제스처 지원

#### **react-intersection-observer 9.16.0**
- **사용 목적**: 뷰포트 가시성 감지 및 지연 로딩
- **적용 위치**: 
  - 무한 스크롤 또는 이미지 지연 로딩 최적화
  - 뷰포트 진입 시 애니메이션 트리거

#### **CSS Modules**
- **사용 목적**: 컴포넌트별 스타일 격리 및 유지보수성 향상
- **적용 위치**: 
  - 모든 컴포넌트에 모듈화된 CSS 적용
  - 클래스명 충돌 방지 및 스타일 캡슐화

### **백엔드**

#### **Node.js + Express 5.1.0**
- **사용 목적**: RESTful API 서버 구축
- **적용 위치**: 
  - 게시글 CRUD, 사용자 인증, 파일 업로드 등 모든 API 엔드포인트
  - 미들웨어 체인을 통한 요청 처리 파이프라인 구성

#### **TypeScript**
- **사용 목적**: 백엔드 코드의 타입 안정성 확보
- **적용 위치**: 
  - 모든 라우트, 모델, 미들웨어에 타입 정의
  - Express Request/Response 타입 확장 (`AuthRequest` 인터페이스)

#### **MongoDB + Mongoose 8.13.2**
- **사용 목적**: NoSQL 데이터베이스로 유연한 스키마 관리
- **적용 위치**: 
  - **Post 모델**: 게시글 데이터 (제목, 내용, 카테고리, 썸네일, 조회수 등)
  - **User 모델**: 사용자 정보 (이메일, 비밀번호, 역할, 로그인 상태 등)
  - **Editors 모델**: 에디터 프로필 정보 (이름, 태그라인, 소셜 링크 등)
  - Mongoose 스키마 훅을 활용한 자동 슬러그 생성, 비밀번호 해싱 등

#### **JWT (jsonwebtoken 9.0.2)**
- **사용 목적**: 무상태(stateless) 인증 시스템 구현
- **적용 위치**: 
  - 로그인 시 토큰 발급 및 쿠키에 저장
  - `authenticate` 미들웨어에서 토큰 검증 및 사용자 정보 추출
  - 토큰 만료 시 자동 로그아웃 처리

#### **bcryptjs 3.0.2**
- **사용 목적**: 비밀번호 해싱 및 보안 강화
- **적용 위치**: 
  - 회원가입 및 비밀번호 변경 시 비밀번호 해싱
  - 로그인 시 입력 비밀번호와 해시된 비밀번호 비교
  - Mongoose `pre('save')` 훅에서 자동 해싱 처리

#### **Cloudinary 2.6.0**
- **사용 목적**: 이미지 업로드 및 CDN 서비스
- **적용 위치**: 
  - 게시글 썸네일 및 에디터 이미지 업로드
  - 이미지 최적화 및 변환 (리사이징, 포맷 변환 등)
  - 서버 저장 공간 부담 감소

#### **Multer 1.4.5-lts.2**
- **사용 목적**: 멀티파트 폼 데이터(파일 업로드) 처리

#### **Nodemailer 6.10.1**

## 🎯 주요 기능 및 구현 방식

### **1. 인증 및 권한 관리**
- **JWT 기반 인증**: 쿠키에 토큰 저장, `authenticate` 미들웨어로 보호된 라우트 접근 제어
- **역할 기반 접근 제어(RBAC)**: `superman`, `ironman`, `human` 3단계 권한 시스템
<!-- - **비밀번호 재설정**: 토큰 기반 이메일 인증 플로우 -->
- **로그인 상태 관리**: `isLoggedIn` 플래그와 스케줄러를 통한 유휴 세션 정리

### **2. 게시글 관리 시스템**
- **카테고리 및 서브카테고리 필터링**: 동적 라우팅으로 카테고리별 게시글 조회
- **슬러그 기반 URL**: SEO 친화적인 URL 구조 (`category-subCategory-6자리숫자`)
- **특별호(Featured) 게시글**: 메인 페이지에 특별히 노출되는 게시글 관리
- **조회수 추적**: 게시글 조회 시 자동 카운트 증가

### **3. 에디터 프로필 시스템**
- **에디터별 게시글 목록**: 에디터 프로필 페이지에서 해당 에디터의 모든 게시글 조회
- **소셜 링크 관리**: 인스타그램, 블로그 등 다양한 플랫폼 링크 연결
- **자동 증가 슬러그**: Counter 컬렉션을 활용한 에디터 고유 번호 생성

### **4. 이미지 관리**
- **Cloudinary 통합**: 이미지 업로드, 최적화, CDN 배포
- **썸네일 자동 생성**: 게시글 썸네일 이미지 관리
- **에디터 프로필 이미지**: 에디터별 프로필 사진 관리

### **5. 사용자 경험 최적화**
- **서버 컴포넌트 활용**: 초기 로딩 성능 향상
- **Suspense 및 스트리밍**: 점진적 콘텐츠 로딩으로 사용자 경험 개선
- **스켈레톤 UI**: 로딩 중 상태 표시
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 환경 지원

### **6. 보안 기능**
- **XSS 방지**: DOMPurify를 통한 HTML 콘텐츠 정제
- **비밀번호 해싱**: bcrypt를 통한 안전한 비밀번호 저장
- **HTTP 보안 헤더**: Helmet을 통한 다양한 보안 헤더 설정
- **CORS 정책**: 허용된 도메인만 API 접근 가능

## 📁 프로젝트 구조

```
taest-mag/
├── frontend/          # Next.js 프론트엔드
│   ├── src/
│   │   ├── app/      # Next.js App Router
│   │   ├── components/  # 재사용 가능한 컴포넌트
│   │   ├── lib/      # API 클라이언트
│   │   ├── store/    # Zustand 상태 관리
│   │   └── utils/    # 유틸리티 함수
│   └── public/       # 정적 파일
│
└── backend/          # Express 백엔드
    ├── src/
    │   ├── routes/   # API 라우트
    │   ├── models/   # Mongoose 모델
    │   ├── middleware/  # 인증/인가 미들웨어
    │   ├── utils/    # 유틸리티 (Cloudinary, Email 등)
    │   └── scheduler/  # 스케줄링 작업
    └── dist/         # TypeScript 컴파일 결과물
```
